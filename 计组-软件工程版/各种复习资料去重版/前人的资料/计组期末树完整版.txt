第三章 计算机功能和互联的顶层视图
    计算机功能和连接
        顶层来看计算机有的功能
            指令的取址和执行

            IO（能直接与cpu交换数据）（某些情况下能直接与内存交换数据：这些操作称为:直接存储器访问DMA）
        顶层来看计算机的连接(各部分怎么连接)
          **总线概念
                干嘛的
                   连接两个或多个设备的通信通路。总线的关键性特征是共享传输介质
		有几种
		    系统总线：连接计算机主要部件的总线。
 			数据总线：提供系统模块中传送数据的路径（线的数目决定能同时传送多少位，决定系统总体性能的关键因素）
			地址总线：指定数据总线上数据的来源和去向（决定能使用最大的存储器容量）
			控制总线：控制对数据地址总线的存取和使用
		决定系统哪些性能和参数
			数据总线：线的数目决定能同时传送多少位；决定系统总体性能的关键因素；寄存器位数；内存单次出的数据
			地址总线：决定能使用最大的存储器容量；决定了可访问单元的个数
			控制总线：控制信号发送命令和时序信号
		方向性
		      ？地址（cpu->各种总线）
		        数（双向）
			控制总线：双向	
			
                总线种类：？数据总线（方向：CPU->各个总线），地址总线（双向的），控制总线（双向的）
            总线仲裁
                数据总线地址总线一般要抢，控制一般不用抢
                方法
                  **1.集中式（搞懂搞透）
                        菊花链式（串式链接的轮循机制）（The chaining method）
                            特点：典型的一个请求线连起所有设备。
				  前面设备响应,后面不会响应。
				  离控制部件越近优先级越高。
				  若是前面设备与控制部件的连接断掉，后面会连不上。
                            优点：有优先级
                            缺点：不公平
                        计数机制（轮询机制？））（The counter polling method）
                            实现方法：时间取模机制
                            优点：公平
                            缺点：慢
                        ？集中控制机制（在cpu判定谁更优）
			独立请求方式（Independent meothod）
                            特点：有一个判定逻辑，判定谁优先
                    2.分布式
			特点：
			    没有中央控制器，而是在每个模块中包含访问控制逻辑，这些模块共同作用

第四五六章总概
    存储体系结构
        1.内外有别 寄存器 -> Cache -> 内存 -> 硬盘 -> 光盘 -> 磁带
        概念：internal memory , main memory
    访问模式
        寄存器以字word为单位
        寄存器以块block为单位
        内存访存以word为单位
        磁盘以扇区block为单位
    访问方法
        随机访问模式 内存，寄存器
            特点
                1.按地址
                2.所有单元时间都相等
        直接访问方式 磁盘，光盘
            特点
                1.按地址
                2.单元访问时间不同
        顺序访问方式 磁带
            特点
                1.不按地址，从头读到需要寻找的地方
    各自性能
